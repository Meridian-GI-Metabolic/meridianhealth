Stage 08 Setup Checklist
MH — Pre-Consultation Reminder

Reference document for admin completing the Brevo UI setup  ·  Full spec: _docs/brevo-automation-flow.html Section 3

What this covers: Creating the Email 08 template in Brevo, building the date-based automation, and running the live test to confirm the 72-hour trigger fires correctly. Complete all three phases in order.
Prerequisite — IP whitelist: If you intend to manage this via the Brevo API, the Replit server IP must first be whitelisted in Brevo (Automation → Security → Authorised IPs). This is a separate task. For UI-only setup, no whitelist change is needed — proceed directly below.

Phase 1 — Create the Email 08 Template in Brevo

Admin note before starting: Confirm the suite/level number for the Brenan Place clinic address with the practice manager before creating this template. The address must match exactly what appears in Email 02 (consultation confirmation). Update both templates at the same time.
  1. 1
    Log into Brevo → Email → Templates. Click Create a new template.
  2. 2
    Set template name: MH — Email 08 — Pre-Consultation Reminder
    Set subject line: Upcoming consultation reminder — Meridian Health
    Set sender name: Meridian Health   Sender email: your verified Meridian sending address.
  3. 3
    Open the drag-and-drop editor. Paste the body copy from Phase 1 — Verbatim Copy below. Use a single-column layout with the Meridian header/footer matching Email 02.
  4. 4
    Conditional Block A — Face-to-face only:
    Add a visibility condition on the block: Contact has tag face_to_face.
    Paste the address content from Block A below into this block.
  5. 5
    Conditional Block B — Telehealth only:
    Add a separate block with visibility condition: Contact has tag telehealth.
    Paste the Coviu instructions from Block B below into this block.
  6. 6
    Verify all merge tags are present and correctly spelled: {{ contact.FIRSTNAME }}, {{ contact.CLINICIAN }}, {{ contact.APPOINTMENT_DATE }}, {{ contact.APPOINTMENT_TIME }}.
  7. 7
    Send a test email to yourself. Confirm layout, merge tag placeholders, and that both conditional blocks appear in the preview (you will need to toggle which tag is applied to see each variant).
  8. 8
    Save and activate the template.

Phase 2 — Build the Brevo Automation

  1. 1
    Go to Automation → Create a workflow → Date-based automation.
  2. 2
    Name the workflow: MH — Pre-Consultation Reminder
  3. 3
    Start condition (trigger):
    Select attribute: appointment_datetime
    Offset: exactly 72 hours (3 days) before
    Do not set a fixed daily send time — Brevo fires at the precise datetime offset, preserving the exact 72-hour lead regardless of when the appointment falls.
  4. 4
    Entry condition filter (both must be true):
    — Contact has tag booked_consult
    — Attribute appointment_datetime is populated / not empty
    This prevents the automation firing for contacts without a confirmed booking.
  5. 5
    Add a Send Email step. Select template: MH — Email 08 — Pre-Consultation Reminder.
  6. 6
    Add a parallel Send SMS step (immediately after, or in parallel branch).
    Wire to: contact.SMS
    Paste the SMS body below.
  7. 7
    Set re-entry: Allow re-entry — a patient may book multiple appointments over time. Each updated appointment_datetime should trigger a fresh reminder cycle.
  8. 8
    Paste the automation description below into the Brevo automation description field (for operational reference).
  9. 9
    Do not activate yet — complete the live test in Phase 3 first, then activate.

SMS 08 — Message Body

Reminder: your Meridian consultation is on {{ contact.APPOINTMENT_DATE }} at {{ contact.APPOINTMENT_TIME }}. Please ensure your forms are completed before your appointment. Questions? Call (03) 9416 4418.

Automation Description (paste into Brevo)

Date-based automation. Fires 72 hours before appointment_datetime. Sends reminder email (MH — Email 08 — Pre-Consultation Reminder) and SMS to prompt form completion. Entry condition: booked_consult tag AND appointment_datetime populated. Re-entry allowed for repeat appointments. Conditional blocks: face_to_face / telehealth — same as Email 02.

Phase 3 — Live Test

Complete this test before activating the automation for real patients. The date-based trigger requires waiting ~72 hours after setup — plan accordingly.
  1. 1
    Create a test contact in Brevo using a real email address and mobile number you control (use clinic staff email/mobile — do not use patient details).
  2. 2
    Apply the tag booked_consult to the test contact.
  3. 3
    Set appointment_datetime on the test contact to exactly 73–74 hours from now (the extra buffer allows for processing time).
  4. 4
    Populate the following attributes on the test contact:
    — FIRSTNAME: any first name
    — CLINICIAN: e.g. A/Prof Michael Hii
    — APPOINTMENT_DATE: human-readable date matching your set datetime
    — APPOINTMENT_TIME: matching time
    — Apply either face_to_face OR telehealth tag (test both variants separately)
  5. 5
    Activate the automation in Brevo now — the automation must be Active for the trigger to fire.
  6. 6
    Wait for the trigger time (~72 hrs before the set appointment). Check your email and mobile for the reminder.
  7. 7
    Verify email received:
    — All merge tags resolved (FIRSTNAME, CLINICIAN, APPOINTMENT_DATE, APPOINTMENT_TIME)
    — Correct conditional block displayed (face-to-face address OR Coviu instructions — not both)
    — Subject line: Upcoming consultation reminder — Meridian Health
  8. 8
    Verify SMS received: date and time display correctly in the message body.
  9. 9
    Check Automation → History in Brevo. Confirm the test contact appears with status Completed.
  10. 10
    Repeat steps 1–9 for the other appointment type (face-to-face or telehealth — whichever wasn't tested first).
  11. 11
    Delete the test contact after confirming both variants.
  12. 12
    Complete the test record table below and update the equivalent table in _docs/brevo-automation-flow.html Section 3.
