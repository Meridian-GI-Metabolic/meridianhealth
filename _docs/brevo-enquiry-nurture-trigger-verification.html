Enquiry Nurture Trigger Verification
MH — Enquiry Nurture · Entry Trigger & Stage 01 Live Test

Reference: _docs/brevo-automation-flow.html Section 1  ·  Code: netlify/functions/enquiry.js

Purpose of this checklist: Confirm that (1) the MH — Enquiry Nurture automation in Brevo is configured with the correct entry trigger, and (2) a live form submission causes Stage 01 to fire and the contact to be correctly tagged within 60 seconds. Complete all three phases in order before signing off.

Phase 2

Brevo Automation Config — Manual Verification

Log into the Brevo dashboard and verify the following settings for the MH — Enquiry Nurture automation. Check each item and tick it off.

  • Navigate to Brevo → Automations. Locate the automation named exactly MH — Enquiry Nurture. Confirm it exists.
  • Open the automation. Inspect the entry trigger. Confirm it is set to:
    Trigger type: Marketing Automation event  ·  Event name: enquiry_submitted
    It must not be set to a Brevo form submission, a list addition, or any other trigger type.
  • Confirm the automation status is Active (not paused or draft). The toggle should show green/active.
  • Confirm the first action node after the trigger is Send email using template MH — Email 01 — Enquiry Follow-Up with no delay (send immediately).
  • Confirm the global exit condition is set: contact has tag booked_consult. This should appear at every wait/condition node or as the automation-level exit condition.
  • Confirm the contact attribute enquiry_submitted_date exists in Brevo → Contacts → Settings → Contact Attributes with type Date.

Phase 3

Live Trigger Test — Form Submission & 60-Second Confirmation

Use a real email address you can check immediately. Stage 01 should arrive within 60 seconds of submission. Have your inbox open before submitting the form. Do not use an address that is already enrolled in the automation — use a fresh test address or one that has never been in this automation.
  • 1
    Open the live Meridian Health website enquiry form. Note the exact time of submission (HH:MM:SS):
    Submission time: ___________
  • 2
    Complete the form using a real, accessible test email address. Select a specific pathway (e.g. bariatric) — do not choose "Unsure" as this does not apply a tag and is harder to verify.
    Test email used: ___________  Pathway selected: ___________
  • 3
    Submit the form. Confirm the site shows a success message (no error displayed to user).
  • 4
    Check the test inbox. Stage 01 email should arrive within 60 seconds.
    Subject line expected: Meridian Health — We've received your enquiry
    Email received at: ___________   Elapsed seconds: ___________
  • 5
    In Brevo → Contacts, search for the test email address. Confirm the contact record shows:
    • Attribute enquiry_submitted_date = today's date
    • Attribute ENQUIRY_PATHWAY = the pathway selected
    • Tag applied = enquiry_bariatric (or whichever pathway was chosen)
  • 6
    In Brevo → Automations → MH — Enquiry Nurture → Logs, confirm the test contact appears in the automation log with Stage 01 marked as sent.

Notes / issues
If Stage 01 does not arrive within 60 seconds: Check Brevo → Automations → Logs for the test contact. Common causes: (1) automation is not active, (2) entry trigger event name does not exactly match enquiry_submitted (check for typos or spaces), (3) the BREVO_API_KEY Netlify environment variable is not set or has expired, (4) the Marketing Automation tracking endpoint is being blocked — check Netlify function logs for Brevo trackEvent error messages.
