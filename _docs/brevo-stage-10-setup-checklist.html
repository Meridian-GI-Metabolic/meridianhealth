Stage 10 Setup Checklist
MH — Pathway Nurture Tracks

Reference document for admin completing the Brevo UI setup  ·  Full spec: brevo-automation-flow.html Section 5  ·  Prerequisite: Stage 09 (Post-Consultation) must be live

This document is the setup checklist for Stage 10, the five pathway-specific nurture automations. It is referenced from the Brevo Activation Checklist. Complete each phase in order. Do not activate any pathway automation until its live test is confirmed.

What this covers: Pushing the 15 Stage 10 email templates to Brevo via the push script, building the five attribute-triggered automations in the Brevo UI, and running a live test on Track 10A (Sleeve Gastrectomy) before activating any track for real patients. The same test procedure applies to each subsequent track.
Body copy status — complete: All 15 emails (3 per track) are finalised and held in scripts/brevo-push-templates.js. Run the push script locally (whitelisted IP required) to create or update the templates in Brevo. See _docs/brevo-push-script-local-run.md for local run instructions. Phase 0 below records the sign-off.
Prerequisites: Stage 09 (Post-Consultation) must already be live. The pathway contact attribute must be created in Brevo before beginning. See brevo-automation-flow.html Section C for the full attribute and tag list. Valid pathway values: sleeve, bypass, reflux_uppergi, glp1_metabolic, revision.

Phase 1 — Create the Five Email Templates in Brevo

The push script handles template content for all 15 emails in one run. Complete the push script step first, then verify each template in the Brevo UI before building automations.

Template names (set by the push script): Template names are defined in scripts/brevo-push-templates.js and match exactly what the push script sends to Brevo. Do not rename templates in the Brevo UI — the script matches by name. See the track cards above for the exact name of each of the 15 templates.
  1. 1
    Run the push script locally to create all 15 Stage 10 email templates in Brevo in one pass:
    BREVO_API_KEY=your_key node scripts/brevo-push-templates.js
    Your machine's IP must be whitelisted in Brevo → Settings → Security → Authorised IPs before running. See _docs/brevo-push-script-local-run.md for full instructions. The script will report success or failure per template.
  2. 2
    Log into Brevo → Email → Templates and confirm all 15 Stage 10 templates are present, showing the correct names (e.g. MH — Pathway 10A — Email 1 — Sleeve Overview) and are in an Active state.
  3. 3
    For each of the 15 templates, verify:
    — Subject line matches the spec in the track card above
    — {{ contact.FIRSTNAME }} merge tag is present in the email body
    — No broken syntax, raw placeholder text, or visible template errors in the Brevo preview
    — Sender name is Meridian Health with the correct verified sending address
  4. 4
    Send a preview test email from Brevo for at least one template per track (five total). Confirm correct rendering at typical screen widths.

Phase 2 — Build the Five Brevo Automations

Repeat the steps below for each of the five pathway tracks. All five automations follow the same structure — the only differences are the trigger value and the templates selected at each step.

  1. 1
    Go to Automation → Create a workflow → Contact attribute updated (or equivalent attribute-change trigger in your Brevo account).
  2. 2
    Name the workflow using the automation name from the relevant track card above (e.g. MH — Pathway: Sleeve Gastrectomy).
  3. 3
    Entry trigger:
    Trigger type: Contact attribute updated
    Attribute: pathway
    Condition: attribute value equals the track's entry value (e.g. sleeve for Track 10A — see track cards above for each track's value)
    This attribute is set by surgeon or admin following the post-consultation review.
  4. 4
    Add three Send Email steps in sequence:
    — Step 1: Select Email 1 template for this track  ·  Timing: Immediately
    — Step 2: Select Email 2 template for this track  ·  Timing: Wait as specified for the track (e.g. +3 days for Sleeve, +4 days for Reflux/Upper GI — see track cards)
    — Step 3: Select Email 3 template for this track  ·  Timing: Wait as specified for the track (e.g. +7 days for Sleeve, +10 days for Reflux/Upper GI)
  5. 5
    Set re-entry: Do not allow re-entry — a patient's pathway is set once by the clinical team. If a pathway needs to be changed, admin should manage this directly and re-trigger manually if appropriate. Allowing re-entry would risk duplicate sends if the attribute is updated for any reason.
  6. 6
    Paste the automation description below into the Brevo automation description field (personalise the track name and entry value for each automation).
  7. 7
    Do not activate yet — complete the live test in Phase 3 first, then activate. Build all five automations in draft before activating any.

Automation Description Template (paste into Brevo — update track name and value)

Triggers when pathway attribute is set to [TRACK VALUE] (e.g. sleeve) by surgeon or admin following post-consultation review. Sends three pathway-specific nurture emails: immediately / +[X] days / +[Y] days. Body copy approved by content team [DATE]. Re-entry disabled — pathway is set once. This automation operates independently of Stage 09. Do not modify wait times without updating _docs/brevo-automation-flow.html Section 5.

Phase 3 — Live Test (Sleeve Gastrectomy Track — 10A)

Complete this test before activating any pathway automation for real patients. Test Track 10A (Sleeve Gastrectomy) first as the reference validation. The same procedure applies to each subsequent track. Use a staff email address — not a patient address.
Use clinic staff contact details only. Do not create test contacts using patient names, emails, or mobile numbers. Use a staff inbox email address only.

Track 10A Live Test — Sleeve Gastrectomy

  1. 1
    Create a test contact in Brevo using a real email address you control (clinic staff inbox — not a patient address).
  2. 2
    Set the following attributes on the test contact:
    — FIRSTNAME: any first name (e.g. Test)
    — CLINICIAN: e.g. A/Prof Michael Hii
    — Do not set pathway yet — setting it is the trigger
  3. 3
    Activate the automation MH — Pathway: Sleeve Gastrectomy in Brevo — it must be Active for the trigger to fire. Go to Automation → MH — Pathway: Sleeve Gastrectomy → Activate.
  4. 4
    Set the pathway attribute on the test contact to sleeve. The automation should fire immediately and send Email 1.
  5. 5
    Verify Email 1 is received. Confirm all of the following:
    — FIRSTNAME merge tag resolved correctly
    — Subject line: Your sleeve gastrectomy pathway — what to expect
    — No broken merge tags, unresolved placeholders, or syntax errors visible in the rendered email
    — Email content is the approved body copy (not a stub or placeholder)
  6. 6
    Check Automation → History in Brevo. Confirm the test contact appears in the workflow and is progressing through the sequence (status: Running at this point — subsequent emails will send at their scheduled wait times).
  7. 7
    Wait for Email 2 (scheduled +3 days from trigger). Confirm it is received and renders correctly. Check Automation History shows the contact progressing.
  8. 8
    Wait for Email 3 (scheduled +7 days from trigger). Confirm it is received and renders correctly. Check Automation History shows the contact reaching status Completed.
  9. 9
    Delete the test contact from Brevo Contacts after confirming all three emails. Search by the staff email used. Confirm deletion before proceeding.
  10. 10
    Confirm MH — Pathway: Sleeve Gastrectomy still shows as Active in Brevo. Do not deactivate.
  11. 11
    Complete the Test Completion Record below and update the matching record in brevo-automation-flow.html Section 5. Both records must be consistent.
Remaining four tracks: Repeat the same live test procedure (steps 1–11 above) for Tracks 10B, 10C, 10D, and 10E. Body copy for all tracks is already finalised. Activate each automation only after its live test is confirmed. Record each test result in the Test Completion Record below.

Phase 4 — Post-Test Sign-Off & Activation Confirmation

Complete this phase only after all live tests have passed for a given track. Confirm test contacts are removed and the automation is cleared for real patient use before informing the practice manager.
  1. 1
    Confirm Automation → History in Brevo shows status Completed for the test contact on each activated track. Take a screenshot for records.
  2. 2
    Confirm the test contact has been deleted from Brevo Contacts for each track tested.
  3. 3
    Confirm each tested automation shows as Active in Brevo Automation.
  4. 4
    Complete the Test Completion Record table below — fill in all rows for each track as tests are completed.
  5. 5
    Update the matching record in brevo-automation-flow.html Section 5 to be consistent with the entries below.
  6. 6
    Inform the practice manager which pathway tracks are now live. Real patients with the matching pathway attribute value set will receive the nurture sequence automatically.
