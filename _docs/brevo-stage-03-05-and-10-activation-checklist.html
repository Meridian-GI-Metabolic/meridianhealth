Brevo Activation Checklist

Stage 03–05 (Enquiry Nurture stubs)  ·  Stage 08 (Pre-Consultation Reminder)  ·  Stage 09 (Post-Consultation)  ·  Stage 10 (Pathway Nurture tracks)

29 May 2026  ·  Internal use — Meridian Health Operations

Overview

This checklist covers the two-phase process for getting Stages 03–05 and Stage 10 live in Brevo:

  1. Phase A — Push template copy via script (requires the Replit IP to be authorised in Brevo)
  2. Phase B — Activate automations via Brevo UI (cannot be done via API; must be done in-browser)
Stage 10 blocker — clinical sign-off required. The 15 Stage 10 pathway emails contain clinical information (supplement regimens, pre-operative protocols, recovery timelines). A clinician or nurse lead must review and sign off all 15 emails before the Stage 10 automations are activated. Use the sign-off form at the end of this document. Do not activate Stage 10 automations until sign-off is recorded.
Script reference: scripts/brevo-push-templates.js — contains all 18 emails as embeddable HTML. Run it with BREVO_API_KEY=… node scripts/brevo-push-templates.js from a machine whose IP is authorised in Brevo.

Phase A  — Push Template Copy via Script

Prerequisite

Authorise the deployment IP in Brevo before running

Brevo requires all API-calling IPs to be explicitly whitelisted. Go to Brevo → Settings → Security → Authorised IPs and add the server IP before running the script.

✓ Step What to do Notes
☐ A1 — Authorise IP In Brevo: Settings → Security → Authorised IPs. Add the IP of the machine that will run the script. For the Replit server this is confirmed in the "Whitelist the deployment IP" task. Without this the script returns a 401 error.
☐ A2 — Confirm stub templates exist In Brevo: Email → Templates. Confirm all 18 stub templates listed below exist with the exact names shown. Create any missing stubs before running the script. Script matches by name — names are case-sensitive.
☐ A3 — Run the script From the project root: BREVO_API_KEY=your_key node scripts/brevo-push-templates.js
The script updates all 18 templates and prints a summary. Review output — any "NOT FOUND" lines mean a stub name doesn't match.		 Script sets isActive: true on all templates as part of the update.
☐ A4 — Run the verification script From the project root:
BREVO_API_KEY=your_key node scripts/brevo-verify-templates.js

The script reads all 18 templates back from Brevo and checks that each subject line matches the manifest and the body contains the clinic phone number. A successful run looks like:

=== Brevo Template Verification ===
Checking 18 templates against manifest…
  ✓ PASS   "MH — Email 03 — Re-Engagement (24hr)"
  ✓ PASS   "MH — Email 04 — Consultation Overview (3 day)"
  … (18 lines total)
──────────────────────────────────
Passed: 18 / 18   Failed: 0 / 18
✓ All 18 templates verified.

If any template fails, the script prints the specific mismatch and exits with a non-zero code. Re-run brevo-push-templates.js to correct content issues, then re-verify. 		Exit code 0 = all pass. Exit code 1 = one or more failures — do not proceed to Phase B until all 18 pass.
☐ A5 — Spot-check in Brevo (optional) As an additional sanity check, open one or two templates in the Brevo editor and confirm body copy is present, subject line is correct, and {{ contact.FIRSTNAME }} merge tags are intact. Optional — the verification script in A4 covers this automatically.

Phase B  — Activate Automations (Brevo UI)

Brevo automations cannot be activated via API. Every step in this phase must be completed in-browser at app.brevo.com → Automations.

B1 — MH — Enquiry Nurture (Stages 03–05)

✓ Step What to do Notes
☐ B1.1 — Open automation Go to Automations → MH — Enquiry Nurture. Open the automation editor.  
☐ B1.2 — Verify Stage 03 step Confirm the +24 hr wait step sends template MH — Email 03 — Re-Engagement (24hr). Check the exit condition: "Contact has tag booked_consult". Exit condition prevents Stage 03 sending to already-booked patients.
☐ B1.3 — Verify Stage 04 step Confirm the +3 day wait step sends template MH — Email 04 — Consultation Overview (3 day). Exit condition present.  
☐ B1.4 — Verify Stage 05 step Confirm the +5 day wait step sends template MH — Email 05 — Pathway Overview (5 day). Exit condition present.  
☐ B1.5 — Activate automation Click Activate. Brevo will validate the flow before activating. Resolve any errors shown. Automation must be in Draft state before activation.
☐ B1.6 — Test contact Use a test contact (staff email) to submit a test enquiry. Confirm: Stage 01 fires immediately, Stage 02 SMS fires at +15 min, then Stage 03 at +24 hr. Check that {{ contact.FIRSTNAME }} resolves correctly. Use a real staff name as the test contact so merge tag is meaningful.

Phase E  — Stage 09 Status Record (Post-Consultation)

Stage 09 (Post-Consultation) is managed separately from Stages 03–05 and Stage 10. Its setup checklist and full test record are in _docs/brevo-stage-09-setup-checklist.html and the matching spec in _docs/brevo-automation-flow.html Section 4. This section records the sign-off status for completeness.
Stage 09 must be live before Stage 10 can function correctly. Stage 09 is the handoff point: it fires after a consultation is marked complete and, once the clinician sets the pathway attribute, triggers the patient's Stage 10 nurture track. Do not activate Stage 10 automations until Stage 09 is confirmed Active.
✓ Step What to do Notes
☐ E1 — Confirm template exists In Brevo: Email → Templates. Confirm the template MH — Email 09 — Post-Consultation Next Steps exists and is active. If not, follow Phase 1 of _docs/brevo-stage-09-setup-checklist.html to create it. Template must be active before the automation can reference it.
☐ E2 — Confirm automation exists and is configured In Brevo: Automations → MH — Post-Consultation. Confirm the entry trigger is Tag consult_completed applied to contact and the single send step references MH — Email 09 — Post-Consultation Next Steps. Full spec: _docs/brevo-automation-flow.html Section 4. If the automation does not exist, follow Phase 2 of _docs/brevo-stage-09-setup-checklist.html.
☐ E3 — Confirm automation is Active Verify the MH — Post-Consultation automation shows status Active in the Brevo Automations list. If still in Draft, click Activate and resolve any validation errors. Automation must be Active before real patients reach Stage 09.
☐ E4 — Confirm live test completed Confirm both test variants were run and passed: (a) test contact with PATHWAY set — conditional pathway block renders; (b) test contact without PATHWAY — conditional block is absent. Full test record in _docs/brevo-stage-09-setup-checklist.html. Both variants must pass before Stage 09 is considered live.
