Stage 09 Setup Checklist
MH — Post-Consultation

Reference document for admin completing the Brevo UI setup  ·  Full spec: brevo-automation-flow.html Section 4  ·  Verbatim copy & setup guide: brevo-email-09-post-consultation.html

This document is the setup checklist for Stage 09, referenced from the Brevo Activation Checklist. It covers creation of the Post-Consultation template, the tag-triggered automation, and the live test. Refer to the activation checklist for the full sequence of pre-launch steps.

What this covers: Creating the Email 09 template in Brevo, building the tag-triggered automation, and running the live test to confirm both variants (with and without PATHWAY set) render correctly. Complete all three phases in order.
Prerequisites: Stage 08 (Pre-Consultation Reminder) must already be live. The consult_completed tag and PATHWAY contact attribute must be created in Brevo before beginning. See brevo-automation-flow.html Section C for the full attribute and tag list.

Phase 1 — Create the Email 09 Template in Brevo

  1. 1
    Log into Brevo → Email → Templates. Click Create a new template.
  2. 2
    Set template name: MH — Email 09 — Post-Consultation Next Steps
    Set subject line: Your next steps with Meridian
    Set sender name: Meridian Health   Sender email: your verified Meridian sending address (e.g. hello@meridianhealth.com.au).
  3. 3
    Open the drag-and-drop editor. Use a single-column layout consistent with Email 02 and Email 08 (Meridian header and footer). Paste the body copy from Phase 1 — Verbatim Copy below.
  4. 4
    Conditional block — pathway reference:
    This block appears only when contact.PATHWAY is set. Wrap the paragraph in Brevo's template syntax within the HTML source view:
    {% if contact.PATHWAY %}
    <p>We look forward to supporting you on your {{ contact.PATHWAY }} journey.</p>
    {% endif %}
    If PATHWAY is not set at send time, this block is silently omitted and the email reads correctly without it.
  5. 5
    Verify all merge tags are present and correctly spelled: {{ contact.FIRSTNAME }}, {{ contact.CLINICIAN }}. Confirm the conditional block syntax is correct: {% if contact.PATHWAY %} … {% endif %}.
  6. 6
    Send a test email to yourself using Brevo's preview/test function. Check that merge tag placeholders are visible in the preview and that the email renders correctly at typical screen widths.
  7. 7
    Save and activate the template. Note the template ID for reference if needed.

Email 09 — Verbatim Copy

Paste exactly as written. Do not rephrase.

Subject line

Your next steps with Meridian

Hi {{ contact.FIRSTNAME }},

Thank you for attending your consultation with {{ contact.CLINICIAN }}. We hope the appointment gave you a clearer picture of your options and the pathway ahead.

Your clinical team will be in touch shortly to outline your recommended next steps and confirm the path forward. In the meantime, please know that you are well supported — our team is here to guide you at every stage of your care.

Conditional block — show only when contact.PATHWAY is set

Brevo template syntax: {% if contact.PATHWAY %} … {% endif %}. If the pathway attribute is not yet set when this email sends, this block is silently omitted.

We look forward to supporting you on your {{ contact.PATHWAY }} journey.

If you have any questions in the meantime, please don't hesitate to contact the clinic on (03) 9416 4418 or reply to this email.

Kind regards,
Meridian Health
Upper GI, Bariatric & Metabolic Care

Required merge tags: contact.FIRSTNAME, contact.CLINICIAN. Optional (within conditional block): contact.PATHWAY. Word count: approximately 95 words excluding conditional block. Confirm FIRSTNAME and CLINICIAN are populated on all contacts before activating.

Phase 2 — Build the Brevo Automation

  1. 1
    Go to Automation → Create a workflow → Tag-based automation (or equivalent entry-trigger option in your Brevo account).
  2. 2
    Name the workflow: MH — Post-Consultation
  3. 3
    Entry trigger:
    Trigger type: Contact tag is added
    Tag: consult_completed
    This tag is applied manually by admin (or via CRM/Genie integration) after the consultation is confirmed as completed.
  4. 4
    Add a Send Email step immediately after the trigger — no wait delay.
    Select template: MH — Email 09 — Post-Consultation Next Steps
    Timing: Immediately
  5. 5
    The automation ends after this single Send Email step. There are no further steps — Stage 10 pathway nurture is triggered independently when the pathway attribute is set by the surgeon or admin post-consultation. Do not chain Stage 10 inside this automation.
  6. 6
    Set re-entry: Allow re-entry — a patient may complete more than one consultation over time. Each application of the consult_completed tag should trigger a fresh send.
  7. 7
    Paste the automation description below into the Brevo automation description field (for operational reference).
  8. 8
    Do not activate yet — complete the live test in Phase 3 first, then activate.

Automation Description (paste into Brevo)

Triggers when consult_completed tag applied by admin. Sends Email 09 (post-consultation next steps) immediately. Confirms the consultation occurred, reassures the patient, and optionally references their pathway if already set (PATHWAY attribute — conditional block). Re-entry allowed for repeat consultations. After this automation exits, surgeon or admin sets the pathway attribute — this triggers the relevant Stage 10 pathway automation independently. Do not chain Stage 10 inside this automation.

Phase 3 — Live Test

Complete this test before activating the automation for real patients. Two variants must be tested: (A) with the PATHWAY attribute set — confirm the conditional block renders; (B) without PATHWAY set — confirm the block is silently omitted and the email reads correctly without it.
Use clinic staff contact details only. Do not create test contacts using patient names, emails, or mobile numbers. Use a staff inbox email address and, if testing delivery, a staff mobile number.

Variant A — With PATHWAY set (conditional block visible)

  1. 1
    Create a test contact in Brevo using a real email address you control (clinic staff inbox — not a patient address).
  2. 2
    Set the following attributes on the test contact:
    — FIRSTNAME: any first name (e.g. Test)
    — CLINICIAN: e.g. A/Prof Michael Hii
    — PATHWAY: any valid value (e.g. sleeve)
  3. 3
    Activate the automation in Brevo — the automation must be Active for the trigger to fire. Go to Automation → MH — Post-Consultation → Activate.
  4. 4
    Apply the tag consult_completed to the test contact. The automation should fire immediately.
  5. 5
    Verify the email is received. Confirm all of the following:
    — FIRSTNAME and CLINICIAN merge tags resolved correctly
    — The conditional pathway block is visible (e.g. "We look forward to supporting you on your sleeve journey.")
    — Subject line: Your next steps with Meridian
    — No broken merge tags or syntax errors visible in the rendered email
  6. 6
    Check Automation → History in Brevo. Confirm the test contact appears with status Completed.

Variant B — Without PATHWAY set (conditional block hidden)

  1. 1
    Create a second test contact in Brevo using the same staff email (or a different staff address).
  2. 2
    Set the following attributes on the test contact:
    — FIRSTNAME: any first name
    — CLINICIAN: any clinician name
    — Do not set PATHWAY — leave it empty / unset.
  3. 3
    Apply the tag consult_completed to this test contact.
  4. 4
    Verify the email is received. Confirm all of the following:
    — FIRSTNAME and CLINICIAN merge tags resolved correctly
    — The conditional pathway block is absent (the email ends after "our team is here to guide you at every stage of your care.")
    — The email reads naturally and completely without the pathway sentence — no blank line or awkward gap where the block would have been
    — Subject line: Your next steps with Meridian
  5. 5
    Check Automation → History in Brevo. Confirm this test contact also shows status Completed.
  6. 6
    Delete both test contacts from Brevo Contacts after confirming both variants. Search by the staff email(s) used. Confirm deletion before proceeding.
  7. 7
    Confirm the automation (MH — Post-Consultation) still shows as Active in Brevo. Do not deactivate — it must remain Active for real patients.
  8. 8
    Complete the Test Completion Record below and update the matching record in brevo-automation-flow.html Section 4. Both records must be consistent.

Phase 4 — Post-Test Sign-Off & Activation Confirmation

Complete this phase only after both Variant A and Variant B tests have passed. The automation is already Active in Brevo (activated in Phase 3, step 3). This phase confirms the test contacts are removed and the automation is cleared for real patient use.
  1. 1
    Confirm Automation → History in Brevo shows status Completed for both test contacts (Variant A and Variant B). Take a screenshot of each for your records.
  2. 2
    Confirm both test contacts have been deleted from Brevo Contacts.
  3. 3
    Confirm the automation (MH — Post-Consultation) shows as Active in Brevo Automation.
  4. 4
    Complete the Test Completion Record table below — fill in all rows.
  5. 5
    Update the matching record in brevo-automation-flow.html Section 4 to match the entries below. Both records must be consistent.
  6. 6
    Inform the practice manager that Stage 09 is live. Real patients with the consult_completed tag applied will now receive Email 09 automatically.
  7. 7
    Note: Stage 10 pathway nurture tracks are separate automations triggered by the pathway attribute. They are not activated by Stage 09 and must be separately configured and activated when copy is complete.
