1   How to Use This Document

This document contains the verbatim body copy and full Brevo setup instructions for Email 09 — Post-Consultation Next Steps. This email sends immediately when the consult_completed tag is applied to a contact and bridges the patient to their Stage 10 pathway nurture track.

Before you begin: Create the Brevo template first (Section 2), then wire it into the automation (Section 3), then test (Section 4). Do not activate the automation until testing is confirmed.

2   Email 09 — Verbatim Copy

Paste this body copy into the Brevo template editor as MH — Email 09 — Post-Consultation Next Steps. The conditional block (dashed box) uses Brevo's {% if %} syntax to optionally reference the patient's pathway if the PATHWAY attribute is already set at send time.

Subject line

Your next steps with Meridian

Hi {{ contact.FIRSTNAME }},

Thank you for attending your consultation with {{ contact.CLINICIAN }}. We hope the appointment gave you a clearer picture of your options and the pathway ahead.

Your clinical team will be in touch shortly to outline your recommended next steps and confirm the path forward. In the meantime, please know that you are well supported — our team is here to guide you at every stage of your care.

Conditional block — show only if contact.PATHWAY is set

Brevo syntax: {% if contact.PATHWAY %} … {% endif %}. If the pathway attribute has not yet been set when this email sends, this block is hidden and the email remains complete without it.

We look forward to supporting you on your {{ contact.PATHWAY }} journey.

If you have any questions in the meantime, please don't hesitate to contact the clinic on (03) 9416 4418 or reply to this email.

Kind regards,
Meridian Health
Upper GI, Bariatric & Metabolic Care

Required merge tags: contact.FIRSTNAME, contact.CLINICIAN. Optional: contact.PATHWAY (within conditional block). Word count: approximately 95 words (excluding conditional block). Confirm these attributes are populated before activating the automation.

3   Brevo Template Setup

3.1 Create the email template

  1. In Brevo, go to Email → Templates → New Template.
  2. Enter the template name: MH — Email 09 — Post-Consultation Next Steps
  3. Subject line: Your next steps with Meridian
  4. Sender name: Meridian Health  ·  Sender email: hello@meridianhealth.com.au (or the verified sender address in your account)
  5. Choose the drag-and-drop editor or paste HTML directly (see Section 3.2 below).
  6. Paste the body copy from Section 2 above, following the conditional block instructions.
  7. Click Save & Quit. The template does not need to be "active" — templates are referenced by name from within automations.

3.2 Conditional block — Brevo syntax

To include the optional pathway reference, wrap the conditional paragraph in Brevo's templating syntax within the HTML source of the template:

{% if contact.PATHWAY %}
<p>We look forward to supporting you on your {{ contact.PATHWAY }} journey.</p>
{% endif %}

The PATHWAY attribute must be created as a contact attribute in Brevo (Settings → Contacts → Contact attributes) before this merge tag will resolve. If it is not set on a contact at send time, the block is silently omitted.

4   Wiring into the MH — Post-Consultation Automation

4.1 Create or open the automation

  1. In Brevo, go to Automation → Create an automation (or open the existing MH — Post-Consultation automation if already created).
  2. Automation name: MH — Post-Consultation

4.2 Set the entry trigger

  1. Entry trigger type: Contact tag is added
  2. Tag name: consult_completed
  3. This tag is applied manually by admin (or via CRM integration) once the consultation is confirmed as completed.

4.3 Add the email send step

  1. Add a Send Email step immediately after the trigger (no wait delay).
  2. Select template: MH — Email 09 — Post-Consultation Next Steps
  3. Timing: Immediately

4.4 End the automation

After the single Send Email step, the automation ends. There are no further steps — Stage 10 pathway nurture is triggered independently when the pathway attribute is set on the contact by the surgeon or admin post-consultation.

Important — handoff to Stage 10: Stage 09 and Stage 10 operate as two separate automations. Email 09 sends on the consult_completed tag. Stage 10 pathway emails trigger when the pathway attribute is set. Both may fire in quick succession but are independent. Do not attempt to chain them within the same automation.

4.5 Automation description (paste into Brevo)

Triggers when consult_completed tag applied by admin. Sends Email 09 (post-consultation next steps) immediately. Confirms the consultation occurred, reassures the patient, and optionally references their pathway if already set. After this automation exits, surgeon or admin sets the pathway attribute — this triggers the relevant Stage 10 pathway automation independently.

5   Testing

How to test Email 09 before going live:

1. Create a test contact in Brevo (use a real email address you control).
2. Set contact.FIRSTNAME and contact.CLINICIAN attributes on the test contact.
3. Optionally set the pathway attribute to any value (e.g. sleeve) to test the conditional block renders correctly.
4. Confirm the MH — Post-Consultation automation is Active in Brevo.
5. Apply the tag consult_completed to the test contact.
6. Confirm the email arrives within a few minutes. Verify:
    a. Merge tags resolve correctly (FIRSTNAME, CLINICIAN).
    b. Conditional pathway block appears when PATHWAY is set, and is absent when it is not.
7. Check Brevo's Automation logs (Automation → History) to confirm the contact entered and exited successfully.
8. Delete the test contact after confirming delivery.
