What needs to happen next

Stages 03–05 — Pending copy

These three stages are built as stubs inside the MH — Enquiry Nurture automation in Brevo. The automation structure and timing are in place; the email body copy has not yet been written and approved. Once the content team delivers approved copy for each stage, it can be entered into the corresponding Brevo template and activated — no further Brevo configuration is required beyond copy entry and a live test for each.

Subject lines specified in the flow document: Following your enquiry with Meridian Health (Stage 03) · What to expect from an initial consultation (Stage 04) · Upper GI, Bariatric & Metabolic care pathways (Stage 05).

Stage 10 — Not yet built (all five tracks)

Email copy for all five pathway tracks (10A–10E) has been authored and is ready to push to Brevo once the push script is run locally from a machine with a whitelisted IP. After the push, the five Brevo automations must be built in the UI and each track tested before activation.

Remaining steps in order:

  1. Confirm copy sign-off with treating clinicians (all five tracks).
  2. Whitelist the sending machine's IP in Brevo → Settings → Security → Authorised IPs.
  3. Run scripts/brevo-push-templates.js locally to push all 15 Stage 10 templates to Brevo.
  4. Build the five pathway automations in Brevo UI (attribute-based trigger: pathway = track value). Full procedure: brevo-stage-10-setup-checklist.html.
  5. Live-test Track 10A (Sleeve) first as a reference, then test Tracks 10B–10E in sequence.
  6. Activate each track only after its live test is confirmed and recorded.
Clinical sign-off required: Stage 10 copy has been authored but has not yet received formal clinical review. Confirm copy for each track is clinically appropriate before pushing templates to Brevo and activating for real patients.
